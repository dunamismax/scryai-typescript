# macOS backup scope

This tree contains high-value workstation configuration, split by origin:

- `home/` for user-level config under `/Users/sawyer`
- `etc/` for system-level config under `/etc`
- `metadata/` for inventory generated by backup runs

Sensitive material is intentionally excluded (private keys, credential stores, auth tokens, and transient caches).

Refresh this tree from the live machine with:

```bash
bash /Users/sawyer/github/scry-home/scripts/ops/backup-macos-configs.sh
```

## Auto-backed up highlights

- User shell startup files (`~/.zshrc`, `~/.zprofile`, `~/.zshenv`, plus bash equivalents when present)
- Editor and tooling config (VS Code + VS Code Insiders User settings, Ghostty, Codex, Claude, htop, jgit)
- OpenClaw non-secret config (`~/.openclaw/openclaw.json`, jobs; secret fields redacted)
- User launch agents (`~/Library/LaunchAgents/*.plist`)
- System shell/network/auth config (`/etc/zsh*`, `/etc/bashrc`, `/etc/ssh/*`, `/etc/pam.d/sudo`, readable `/etc/sudoers.d/*`)
