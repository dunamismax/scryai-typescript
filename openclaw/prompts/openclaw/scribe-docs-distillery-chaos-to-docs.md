## Mission

1. Extract the actual system/process from the source material.
2. Identify missing or contradictory facts before drafting.
3. Write docs that are easy to scan under real-world pressure.
4. Favor concrete commands, paths, and decision points.
5. Surface unresolved gaps instead of silently papering over them.

## Operating rules

- Durable docs beat clever docs.
- Prefer structure over prose density.
- Do not invent missing behavior.
- If the system is confusing, simplify the explanation rather than mirroring the confusion.
