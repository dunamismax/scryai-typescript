Goal:
Turn a rough intent into strong first-pass writing that is ready to react to, edit, or send.

Fill these in before running:
- Writing task: [email, post, note, announcement, README section, etc.]
- Audience: [who it is for]
- Purpose: [what it should achieve]
- Constraints: [tone, length, must-include points, must-avoid points]
- Raw notes: [paste the messy material]

## Mission

1. Extract the real message from the rough notes.
2. Write at least three viable versions.
3. Keep them meaningfully different, not cosmetic rewrites.
4. Preserve any required facts, asks, dates, or constraints.
5. End with the strongest recommended version.

## Operating rules

- Cut fluff, fake warmth, and throat-clearing.
- Match tone to audience and stakes.
- If the source material is contradictory, flag it.
- Prefer crisp, sendable copy over ornate prose.

## Required output

# Draft Pack

## 1) Message intent
What this writing needs to accomplish.

## 2) Version A — terse
Fast, sharp, minimal.

## 3) Version B — balanced
Default strong version.

## 4) Version C — bold
A more opinionated or higher-energy variant, if appropriate.

## 5) Best recommendation
Which version you would send and why.

## 6) Risks / missing facts
Anything that still needs confirmation before sending.

## Decision standard

Produce drafts that are good enough to use, not just admire.
