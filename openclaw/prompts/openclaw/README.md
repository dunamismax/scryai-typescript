# OpenClaw Prompt Library

Reusable prompts aligned to the current Discord bench and workspace-channel architecture.

## Current set

### Orchestration / home lane
- `scry-lane-router-and-execution-brief.md` — decide the best lane, agent, scope, and paste-ready handoff.

### Codex
- `bug-hunt-mergeable-fix-scout.md` — find the single best upstream bug to fix next.
- `codex-lab-prototype-spike.md` — run a rough spike and decide whether it deserves a real build lane.
- `codex-shipyard-scoped-implementation.md` — implement a narrow feature/fix with verification.
- `codex-review-desk-adversarial-review.md` — do hard-nosed review before shipping.

### Research
- `research-deep-dive-answerable-memo.md` — produce a serious sourced memo with a clear answer.
- `research-market-radar-opportunity-scan.md` — scan markets, competitors, wedges, and timing.
- `research-kill-or-pursue-decision-memo.md` — force a kill / pursue / park call.

### Scribe
- `scribe-draft-room-draft-pack.md` — generate rough-but-usable first drafts fast.
- `scribe-docs-distillery-chaos-to-docs.md` — turn notes, code, and context into durable docs.

### Operator
- `operator-openclaw-ops-runtime-triage.md` — diagnose OpenClaw runtime, routing, cron, config, and backup issues.
- `operator-automation-yard-automation-builder.md` — design or build scripts, jobs, and automation.
- `operator-ops-watch-incident-triage.md` — produce incident status, containment, and next move.

### Sentinel
- `sentinel-security-audit-prioritized-hardening.md` — run a practical security / hardening pass.

### Luma
- `luma-shot-lab-creative-treatment.md` — build a visual treatment, shot plan, and production approach.

## Notes

- These prompts are written for practical future reuse, not one-off novelty.
- Most files include placeholders like `[PROJECT]`, `[GOAL]`, or `[CONSTRAINTS]`. Replace them before use.
- The library intentionally mirrors the current Discord workspaces so routing stays obvious.
