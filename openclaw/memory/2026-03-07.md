## Discord native command/session labeling fix

- Diagnosed a Discord regression in `~/openclaw`: native slash commands (`/new`, `/reset`) were building inbound context without the same guild/channel session metadata used by normal Discord messages.
- Root cause: `src/discord/monitor/native-command.ts` was not persisting `GroupChannel` / `GroupSpace` and used a weak conversation label, so fresh sessions could fall back to `discord:g-<channelId>` instead of `discord:<guildId>#<channel>`.
- Added a live patch in `~/openclaw` to align Discord native-command metadata with the normal message path and added regression coverage in `src/discord/monitor/native-command.plugin-dispatch.test.ts` for bound guild-channel slash commands.
- Restarted the OpenClaw gateway after patching so the Discord fix is loaded in the running instance.
- Config audit result: Discord bindings, guild allowlist, native commands, and thread bindings all looked sane; this issue was code-path drift, not a bad Discord config.
